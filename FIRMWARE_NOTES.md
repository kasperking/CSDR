# CSDR Firmware – Developer & User Notes

**MCU:** STM32H750VBT6 (Cortex-M7, 480 MHz) · **Display:** ST7796 480×320 FMC (default) or ST7789 240×320 FMC (compact) · **Audio codec:** WM8731 (SAI I²S) · **Last updated:** 2026-05-28

---

## 1. Hardware Platform

### MCU & Clocks

| Item | Value |
|------|-------|
| MCU | STM32H750VBTX (128 KB flash, 864 KB DTCM/SRAM) |
| SYSCLK | 480 MHz (PLL1: HSE 25 MHz → M=5, N=192, P=2) |
| HSE | 25 MHz crystal |
| SAI1 (audio) | 12.2881 MHz (PLL2: M=2, N=58, P=59) → 48 kHz sample rate |

Clock settings live in `BSP/Inc/hw_config_active.h` (auto-generated by `tools/hw_config.py`). Do **not** edit by hand — regenerate via the tool.

### Panel Select

Panel is chosen at compile time via `BSP/Inc/lcd_panel_config.h`:

```c
#define LCD_PANEL  LCD_PANEL_ST7796   // 480×320, default
// #define LCD_PANEL  LCD_PANEL_ST7789   // 240×320, compact
```

ST7796 uses ~253 KB RAM for line buffers; ST7789 uses ~148 KB.

## 4. Key / Button Behavior

All keys are debounced (20 ms) via `Key_Poll` / `Key_Press` / `Key_PressOrRepeat`. Hold-repeat fires continuously while held (threshold 600 ms, rate 200 ms).

| Key | Menu closed | Menu open |
|-----|-------------|-----------|
| **MENU** | Open menu | Close menu |
| **F1** | Volume −2 (floor 0), repeat | Cursor Up |
| **F2** | Volume +2 (cap 100), repeat | Cursor Down |
| **F3** | *(no action)* | Confirm / execute ACTION item |
| **F4** | *(no action)* | Back / exit menu level |
| **BAND** | Next band (BPF/LPF switched) | *(no action)* |
| **MODE** | Cycle AM→FM→USB→LSB→CW→AM | *(no action)* |
| **PTT (PB12)** | Toggle TX on/off | *(no action)* |
| **ENC↕** | Tune by current step | Menu encoder-edit (item value) |
| **ENC⊙** | Cycle mode | Select item / run ACTION |
| **ENC⊙** *(long)* | Cycle spectrum zoom | *(no action)* |

**Volume setter:** `cat_set_volume(vol)` writes WM8731 HP register (range 90–121 ≙ −31 dB to 0 dB; vol=0 forces mute at register 0x2F) then sets `display_dirty = true`; display updates on the next 200 ms tick.

**F4 in SWR scan:** hold during sweep to abort; press after complete to exit.

**F4 during TX (DIGI toggle):** hold F4 to toggle digital-mode TX (WSJT-X dirty path).

## 5. Spectrum / Waterfall Behavior

### Spectrum (SPEC zone)

- **Source:** `dsp.fft_mag_db[256]` — power values after fftshift, pre-log.
- **Bar graph:** bottom-up per column, 2-tier colour gradient: dark-blue→mid-blue (noise floor) → green→yellow→red (signal).
- **Grid:** horizontal lines at 25/50/75 % height; vertical dots every 40 px.
- **BW markers:** cyan vertical lines at demod filter edges, derived from `bw_hz / sample_rate` ratio scaled to zoom window.
- **Center marker:** 1-px dark | 1-px white | 1-px dark tricolor at DC bin.
- **Peak hold:** 1-px peak line persists until signal drops.
- **TX mode:** SPEC zone replaced by amber compact TX mic spectrum (5 fps); RX SPEC/WF are disabled when `tx_mode == true`.
- **Update rate:** ~28 fps (decoupled from waterfall).
- **Suppressed when menu is open.**

### Waterfall (WF zone)

- **Ring buffer:** `s_wf_buf[WF_H][LCD_W]` (72 rows × 480 columns on ST7796).
- **IIR smoothing:** α=0.72 per bin (`s_wf_smooth[b] = 0.72×prev + 0.28×new`).
- **Level mapping:** `10×log10(pwr) − K dB`, clipped to [−120 dB, −20 dB] → 0–255 index.
- **Colour palette:** 256-entry thermal LUT (black→dark-blue→cyan→green→yellow→white).
- **Scroll:** ring-slot push — single new row written to ring head slot; head pointer decrements. FMC push sends the single new row only (72× bandwidth reduction vs. full-frame).
- **Hardware scroll rejected:** MADCTL landscape mode breaks the scroll axis on these controllers; software ring is the correct approach.
- **Rate cap:** one waterfall line per ~77 ms (~13 fps), enforced in `CSDR_Loop`.
- **Suppressed when menu is open or TX active.**

## 7. S-meter / TX Meter

### RX S-meter

- **Source:** `dsp.signal_power_db` — pre-AGC RMS of `√(filt_i²+filt_q²)` after the FIR LPF; exponentially smoothed α=0.9/0.1.
- **24-stick progressive density bar:** stick density increases as signal rises; colors S1–S6 green, S7–S9 yellow, S9+ red.
- **Bar mapping:** `bars = (signal_db + 73) / 3`, clamped 0–12.
  - bars=0 ≈ −73 dBFS · bars=9 ≈ −46 dBFS · bars=10/11 ≡ S9+20/S9+40
- **Ruler ticks:** hairline scale above sticks (`S 1 3 5 7 9 +20 +40`).
- **BATT voltage:** shown in meter zone as `BATT X.XV`; red when <11.5 V.
- **Update rate:** `SDR_UI_UpdateSMeter`, every 200 ms when menu closed and RX active.

### TX ALC + SWR meter

Active when `g_sdr.tx_mode == true`.

- **ALC bar:** same 12-segment geometry as S-meter; source `g_analog.alc_percent` (0–100 %).
- **SWR text:** label + value, no bar. Green <2.0, yellow 2.0–3.0, red ≥3.0. Source `g_analog.swr_x100 ÷ 100`.

## 8. SWR Scan

**Entry:** Menu → "SWR Scan" (ACTION item, encoder click or F3).  
**Blocking:** occupies SPEC+WF zones; DSP audio continues via DMA interrupts.

| Parameter | Value |
|-----------|-------|
| Span | ±200 kHz around current VFO |
| Step | 10 kHz |
| Max points | 201 |
| TX settle | 30 ms per step |
| SWR floor | 1.00 (hardware min) |

**Sweep sequence per step:**  
1. Set BPF + LPF for step frequency.  
2. `scan_tx_on()` → CLK0 reprogrammed to step × 4, T/R relay asserted.  
3. Wait 30 ms, call `Analog_Update()` twice.  
4. Record `g_analog.swr_x100`.  
5. `scan_tx_off()` → T/R released, CLK0 restored to `centre_hz + lo_offset_hz`.  
6. Incremental plot update.

**Plot layout:**

```
Y= SPEC_Y     Title bar  "SWR SCAN  7.100 MHz  ±200 kHz"
Y= SPEC_Y+14  Plot area  (SWR 5.0 = top, SWR 1.0 = bottom)
...
              Frequency axis (start / centre / end MHz)
              Result bar "MIN 1.35 @ 7.090 MHz  [F4=EXIT]"
```

**Colour:** green <1.5 · yellow 1.5–2.0 · orange 2.0–3.0 · red ≥3.0 · cyan = minimum marker.  
**Abort:** hold F4 during sweep; partial results displayed.  
**Exit:** press F4 after scan complete (requires release then fresh press).  
**Restore:** BPF/LPF reset to `g_sdr.band_idx`; LO restored; `g_sdr.display_dirty = true`.

## 9. DSP Signal Flow

**Sample rate:** 48 kHz · **Block size:** 256 samples · **SAI format:** 16-bit right-justified in 32-bit slots (RX: `(int16_t)(uint16_t)word`; TX: `(int32_t)(int16_t)sample`)  
**FFT:** CMSIS `arm_cfft_f32` 512-point (256-sample input → ±24 kHz), Hann window, conjugated input for correct ±frequency orientation (`SDR_USE_CMSIS_FFT`).

### RX Pipeline (per 256-sample block)

```
SAI DMA int32 I/Q (interleaved)
  ↓  (int16_t)(uint16_t) → float × 1/32767
Pre-mix IIR DC block ×2      H(z) = (1−z⁻¹)/(1−0.995z⁻¹)   removes ADC bias
  ↓
NCO down-mix                 32-bit phase acc, 1024-entry LUT, supports −freq
  ↓
Post-mix IIR DC block ×2     removes QSD LO leakage (centre spike suppression)
  ↓
IQ mismatch correction       Q = (Q − iq_p×I) × iq_g_inv   (Gram-Schmidt)
  ↓
FFT accumulate               Hann window → arm_cfft_f32 → mag² per bin
  ↓
FIR LPF I and Q              64-tap Hann-windowed sinc, odd-tap (linear phase), cutoff = bw_hz/Fs
  ↓
IF-shift NCO mix             programmable IF/RIT shift
  ↓
Demodulate
  AM   → sqrt(I²+Q²)
  FM   → atan2 differentiator + 75 µs de-emphasis IIR
  USB  → (I+Q)/2
  LSB  → (I−Q)/2
  CW   → AM envelope × 700 Hz BFO (32-bit NCO)
  ↓
Audio FIR LPF                32-tap, 4 kHz cutoff
  ↓
Audio IIR DC block           removes AM envelope DC bias
  ↓
AGC                          peak-hold + hang; attack 1 ms always
                             SSB: 200 ms decay, 500 ms hang
                             CW:  75 ms decay,  100 ms hang
                             AM:  500 ms decay,  2 s hang
                             FM/DIGU/DIGL: bypass (unity gain)
  ↓
Scale → int16, stereo L=R → SAI TX DMA (headphone output)
```

**Signal power for S-meter:** pre-AGC (post-FIR) RMS, α=0.9/0.1 smoothing.

**Noise blanker (NB):** Optional time-domain impulse zeroing step-3d on IQ; `DSP_NB_Set()` / `NoiseBlanker_t`; disabled by default.

**Noise reduction (NR):** stub only — not implemented.

### TX Pipeline (per 256-sample block)

```
USB Audio OUT ring (PC → radio, int16 LE stereo, L channel used)
  ↓
Audio IIR DC block
  ↓
TX gain (audio_gain, 0..1)
  ↓
Audio FIR LPF 32-tap 4 kHz  applied before modulation
  ↓
Compressor 4:1 above 0.70    attack 1 ms, release 50 ms
Soft limiter C1-smooth       y = 1 − 0.0025/(|x|−0.90) for |x|>0.95
  ↓
Modulate
  USB/LSB → Hilbert phasing: 63-tap Hamming Hilbert (Q), matched delay (I)
  AM      → 0.5 + 0.5×audio  (DSB+carrier; no suppression)
  FM/CW   → audio × 0.7 on I only  ← placeholder (no phase integrator/deviation)
  ↓
DC block                     pre-FFT centre spike removal for TX display
  ↓
FFT accumulate               → TX mic spectrum (amber, INFO zone, 5 fps)
  ↓
Scale → int16 right-justified → SAI TX DMA → WM8731 DAC → QSE mixer
```

**WSJT-X / DIGI mode:** bypass compressor + use full mic gain; F4-hold toggles. FM/CW TX is AM placeholder — no true FM modulation.

## 10. LO Architecture

### Si5351A + 74LVC74 shared quadrature

```
25 MHz XTAL
    │
Si5351A
    └─ PLL_A  VCO 600–900 MHz
          └─ CLK0  →  4 × RF frequency
                          │
                  74LVC74 dual D flip-flop (Johnson ÷4)
                      ├─ I (0°)  ──┬── QSD I (RX mixer)
                      │            └── QSE I (TX mixer)
                      └─ Q (90°) ──┬── QSD Q (RX mixer)
                                   └── QSE Q (TX mixer)
CLK1, CLK2: powered down · PLL_B: unused
```

RX and TX mixers are fed from the **same** 74LVC74 output. A single CLK0 serves both; no separate TX LO. Phase relationship is exact (hardware Johnson counter, not Si5351 phase-offset registers — avoids even-MS\_div constraint).

**LO offset** (`lo_offset_hz`, default 0 Hz, production ≈18 kHz): Si5351 is programmed to `freq_hz + lo_offset_hz`; NCO shifts baseband by the same amount in the opposite direction.

```
NCO freq = −(lo_offset_hz + if_shift_hz)
```

Keeps QSD/QSE off the exact receive frequency → suppresses LO self-reception at antenna.

**Constraints:** VCO 600–900 MHz · CLK0 ≈100 kHz–150 MHz · XTAL: 25 MHz (overrides si5351.h 27 MHz default at init).

## 11. RF Front-End

### Band-Pass Filter (BPF) — FST3253 analog mux

| S1:S0 | Pins | Filter |
|-------|------|--------|
| 00 | PA4:PA5 | 20/30 m |
| 01 | | 40 m |
| 10 | | 15–10 m |
| 11 | | 80 m |

PA6=OE1 (TX path), PA7=OE2 (RX path) — always complementary.

### Low-Pass Filter (LPF) — 74HC238 3-to-8 decoder

| Y output | Band | PA2:PA1:PA0 |
|----------|------|-------------|
| Y0 | 1.8 MHz (160 m) | 000 |
| Y1 | 3.5 MHz (80 m)  | 001 |
| Y2 | 5.8 MHz (60 m)  | 010 |
| Y3 | 8–17 MHz (40/30/20 m) | 011 |
| Y4 | 17–32 MHz (17–6 m) | 100 |
| Y7 | All off | 111 |

### RX Attenuator — PE4302 6-bit step attenuator

- **Range:** 0–31.5 dB in 0.5 dB steps.
- **Interface:** bit-bang SPI on PC4 (DATA), PC5 (CLK), PB0 (LATCH).
- **Current value:** `g_att.current_atten_db`.

### T/R Switch

Single relay, PB2 GPIO. TX: assert relay + reprogram CLK0 + start codec TX. RX: release relay + restore CLK0.

## 12. Audio Codec (WM8731)

- **Interface:** I2C @ 0x1A<<1 (shared with Si5351 at 0x60).
- **Data:** SAI1 I²S, 16-bit right-justified in 32-bit slots, 48 kHz stereo.
  - SAI1_B RX: I/Q from QSD mixer (L=I, R=Q).
  - SAI1_A TX: demodulated audio to headphone jack (L=R=mono output) / TX IQ to QSE.
- **DMA:** circular ping/pong, block size 256 samples (512-word DMA buffer per direction).
- **HP volume register:** range 90–121 (−31 dB to 0 dB); vol=0 → mute at 0x2F.
- **IMPORTANT:** Do not call `USB_Audio_WriteRX` from the DMA ISR — causes a hard-lock when the host opens the audio stream. Call from main loop only.

## 13. USB Interface

### Composite Device (5 interfaces)

| Interface | Type | EP |
|-----------|------|----|
| AC (Audio Control) | — | — |
| AS_OUT (Audio OUT) | ISO | EP3 OUT |
| AS_IN (Audio IN)  | ISO | EP3 IN |
| CDC_CTRL (CAT control) | Interrupt | EP2 IN |
| CDC_DATA (CAT data) | Bulk | EP1 IN/OUT |

**USB Audio ring:** 48 packets × 192 bytes = 9216 bytes (9 ms). Overrun threshold 83% (7680 B). Underrun prefill: 8 packets on `SetStreaming`. Drift correction: 3-second streak at >20 packets triggers `rx_rd` trim to 8 packets.

### CAT Protocol (Kenwood TS-2000 emulation)

- **Device ID:** `ID019`
- **Transport:** USB CDC VCP, semicolon-terminated ASCII frames, one frame per USB transfer with 2 ms inter-frame pacing (prevents flrig coalescing freeze).
- **AI push:** disabled (AI0 mode).

**IF frame format (after 2026-05-28 P15 fix):**

```
IFxxxxxxxxxxxxxPPPPP+RRRRRRRTT0MVFBS0; 
│               │    │       │││││  │
│               │    │       │││││  └─ ';' terminator
│               │    │       ││││└─── P15 = '0' (frozen, offset 36)
│               │    │       │││└──── split (offset 32)
│               │    │       ││└───── VFO A/B (offset 30)
│               │    │       │└────── mode (offset 29)
│               │    │       └─────── TX/RX (offset 28)
│               │    └─────────────── RIT/XIT offset (7 chars)
│               └──────────────────── step size (5 chars PPPPP)
└──────────────────────────────────── frequency (11 chars)
```

Frame must be **37 data characters** before `;` (38 total). Hamlib 4.7.1 `kenwood_safe_transaction` checks `strlen == if_len` where `if_len = 37` for TS-2000 — short frame → `RIG_EPROTO`. P15 at offset [36] satisfies this. If `CAT_BuildIF` is ever modified, verify the output is still 37 data chars before `;`.

**Implemented CAT commands:**

| Category | Commands |
|----------|----------|
| VFO / Freq | FA, FB, IF, FR, FT, DC |
| Mode | MD |
| TX/RX | TX, RX, TQ |
| Split / RIT | SP, RT, RC, RU, RD, XT, IS |
| S-meter | SM |
| VFO ops | VS, UP, DN, BU, BD |
| AI | AI |
| Info | ID, PS, VV |
| Audio | AG (volume), MG (mic gain) |
| DSP | NR, NB, FW, SH, SL |
| Squelch | SQ |
| ATT / Pre | RA, PA |
| Power | PC |
| Scan | BC, TS |
| Memory | MW, MR |
| Misc | GT, RG, RL, LK, KS, MN, MP, EX, DS, TC, KY |

**Forensic ring:** last 32 transactions logged; opcode hit counters in `usb_cat.c`.

**SET echo consistency:** all 13 SET commands (MD/NR/NB/FW/SH/GT/RA/AG/SQ/PC/RC/RU/RD) echo the full accepted value back. Parser state resets on USB reconnect. 200 ms command timeout.

**BW / mode clamping:** `cat_set_bw()` is mode-aware — AM/FM calls are silently ignored (SH); FM `FW` query returns `0000`.

## 16. Build System

**Toolchain:** GCC ARM (STM32CubeIDE 2.1.1), optimization `-Os`, define `ARM_MATH_CM7`.

**Flash constraint:** 128 KB is tight — heavy stub use for diagnostics and lesser-used CAT commands.

### Hardware config tool

```
python tools/hw_config.py           # interactive wizard
python tools/hw_config.py --apply   # reuse saved answers
```

Generates `BSP/Inc/hw_config_active.h` and patches `Core/Src/main.c`, `stm32h7xx_hal_conf.h`, `system_stm32h7xx.c` for PLL/HSE values.

**Presets:** `hw_test_fmc` (bench, loose FMC timing), `hw_prod_v1` (production), `hw_long_fpc_debug` (long-FPC board).

### Setup workflow after CubeMX regen

1. Generate from `CSDR.ioc`.
2. Copy `BSP/` into project root; add `BSP/Inc` + `BSP/Src` to IDE include/source paths.
3. Apply `PATCH_main.c` template (includes, init calls, loop body, SAI/DMA callbacks).
4. Run `hw_config.py`.
5. **Post-regen checklist:** verify USB composite descriptor intact (5 interfaces); verify `USBD_CDC` calls not restored by CubeMX; verify `HAL_SAI_RxHalfCpltCallback` / `TxHalfCplt` hooks present; verify no `USB_Audio_WriteRX` in DMA ISR.
6. Build.
